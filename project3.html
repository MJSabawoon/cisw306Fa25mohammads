1- Executive Summary

I checked the website Wikipedia.org to see how easy it is for all users to use, including people with disabilities. The website looks well organized and mostly easy to read. The color contrast is good, and text is large enough. However, some menus are hard to use only with a keyboard, and a few images do not have full descriptions.

2- Background about Evaluation

This evaluation was part of Project 3 for CISW 306. The goal was to test how well a website follows web accessibility rules, especially WCAG 2.2 and Section 508. I used simple online tools like WAVE, W3C Validator, and the browser’s Inspect tool to check accessibility.

3- Scope of Review

  1. I reviewed three parts of Wikipedia: the homepage, an article page, and the search page.
  2. I checked if users can read, see, and move through the website easily.
  3. I also looked at how it works on mobile and computer screens.

5- Review Process

  1. I opened the website on my laptop and mobile.
  2. I ran a test with WAVE to find accessibility issues.
  3. I checked if I could move through the page using only the keyboard.
  4. I looked at the color contrast and image descriptions.
  5. I wrote down my results and suggestions.

6- Results and Recommended Actions

Perceivable

  • ✔ Pass: Text and images are clear and easy to see.
  • ❌ Issue: Some pictures don’t have alt text.

    Recommended action: Add meaningful alt attributes to informative images or aria-hidden="true" for decorative images.

Operable

  • ✔ Pass: Most links and buttons work with a keyboard.
  • ❌ Issue: Drop-down menus could be easier to navigate.

    Recommended action: Ensure menus are keyboard-focusable, use visible focus styles, and add proper ARIA roles if needed.

Understandable

  • ✔ Pass: The layout is simple and easy to follow.
  • ❌ Issue: Some pages have too much text and could be clearer.

    Recommended action: Break long text into shorter paragraphs, use headings, and add summary bullets for long pages.

Robust

  • ✔ Pass: The site works well on desktop and mobile.

Overall

Wikipedia is mostly accessible, but it still needs small improvements to fully meet WCAG 2.2 standards.
